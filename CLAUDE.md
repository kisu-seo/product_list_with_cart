# [핵심 원칙]
설명은 이해하게 쉽게 3줄 이하 불릿 형태로 핵심만 설명해 줘.

접근성(A11y) First: 모든 HTML은 스크린 리더가 읽을 수 있도록 접근성을 고려해서 짜줘 (aria-label, 시멘틱 태그 활용).

언어 일관성: 기술적인 키워드를 제외한 모든 가이드와 대화는 한국어로 진행하며, 영어로 답변이 나올 경우 즉시 한국어로 재번역해서 알려줘.

## [기술 스택 강제 지침]

### 1. React (컴포넌트 뼈대 조립 및 데이터 분리)
- **컴포넌트 분리:** 전체 화면을 통짜로 짜지 말고, 유지보수를 위해 좌측/상단의 **상품 목록 컴포넌트(Product List Section)**와 우측/하단의 **장바구니 컴포넌트(Cart Section)**로 명확히 나누어 조립해 줘.
- **데이터 분리 (Data Driven):** 화면에 보이는 디저트 정보(이미지, 이름, 카테고리, 가격)는 컴포넌트 내부에 하드코딩하지 마. 제공된 `data.json` 파일의 배열(Array) 데이터를 불러온 뒤, `.map()` 함수를 사용해 개별 '상품 카드 컴포넌트'로 깔끔하게 렌더링해 줘.

### 2. Tailwind CSS (반응형 레이아웃 및 정밀한 상품 UI)
- **유틸리티 클래스 활용:** 별도의 CSS 파일 작성 없이 **Tailwind CSS** 유틸리티 클래스만 사용하여 시안과 동일한 색감(주황/빨강 포인트 컬러), 텍스트 스타일, 둥근 모서리를 구현해.
- **상품 UI 디테일:** 장바구니에 담긴 상품의 이미지는 붉은색 테두리(`border-red-500` 등)가 생기도록 하고, 기존 'Add to Cart' 버튼이 수량을 조절하는 주황색 '+/- 버튼'으로 자연스럽게 교체되게 만들어 줘.
- **Mobile First (반응형 레이아웃):** 모바일(기본)에서는 상품 목록이 1열로 쭉 나열되고 장바구니가 맨 아래에 위치하는 세로 구조(`flex-col`)로 배치해 줘. 
- **데스크톱 화면 전환:** 데스크톱(`md:` 이상)에서는 화면이 가로로 넓어지며, 좌측은 상품들이 3열 격자(`grid-cols-3`)로 정렬되고 우측은 장바구니가 고정되어 따라다니는 2단 레이아웃으로 매끄럽게 전환되게 해 줘.

### 3. React State & JavaScript (장바구니 상태 관리 및 모달 로직)
- **상태 관리:** `cartItems`(장바구니에 담긴 상품 배열) 상태를 React의 `useState`로 관리해 줘. 배열 안에는 상품의 ID, 이름, 단가, 그리고 **수량(quantity)** 정보가 포함되어야 해.
- **동적 데이터 업데이트:** 사용자가 상품의 '+ / -' 버튼을 누르면 `cartItems` 상태가 업데이트되고, 우측 장바구니 영역의 '총 수량'과 '총 결제액(Order Total)'이 즉시 다시 계산되어 화면에 렌더링되어야 해.
- **빈 장바구니 피드백:** 장바구니에 담긴 상품이 없을 때는 계산 내역 대신 빈 장바구니 일러스트와 "Your added items will appear here"라는 안내 텍스트를 보여줘.
- **모달 창 및 초기화 로직:** 'Confirm Order' 버튼을 누르면 화면 중앙에 반투명한 배경과 함께 주문 완료 모달(Modal) 창이 떠야 해. 모달 창 안의 'Start New Order' 버튼을 클릭하면 장바구니 상태값이 완전히 초기화(빈 배열)되고 모달이 닫히도록 구현해 줘.
